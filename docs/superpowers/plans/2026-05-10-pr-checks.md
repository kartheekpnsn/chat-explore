# PR Checks Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add ruff-based linting/formatting checks and pytest runs to both local pre-commit hooks and a GitHub Actions PR workflow.

**Architecture:** Three files are added/modified — `pyproject.toml` (swap black for ruff+pre-commit in dev deps), `.pre-commit-config.yaml` (two ruff hooks), and `.github/workflows/pr-checks.yml` (CI workflow). No source code changes needed.

**Tech Stack:** Python 3.11, uv, ruff 0.11.12, pre-commit, pytest, GitHub Actions

### Task 2: Add ruff configuration to pyproject.toml

**Files:**
- Modify: `pyproject.toml`

- [ ] **Step 1: Append ruff config section to `pyproject.toml`**

Add the following at the end of `pyproject.toml`:

```toml
[tool.ruff]
target-version = "py311"
line-length = 88

[tool.ruff.lint]
select = ["E", "F", "I"]

[tool.ruff.format]
quote-style = "double"
```

`E` = pycodestyle errors, `F` = pyflakes, `I` = isort. `line-length = 88` matches the former black default.

- [ ] **Step 2: Run ruff check to see current state of the codebase**

```bash
uv run ruff check .
```

Expected: Some violations may appear. Note them — they will need to be fixed in Task 3.

- [ ] **Step 3: Run ruff format to auto-format the codebase**

```bash
uv run ruff format .
```

Expected: ruff reformats files. Output lists files changed.

- [ ] **Step 4: Run ruff check with auto-fix for fixable lint issues**

```bash
uv run ruff check . --fix
```

Expected: Fixable issues (like import ordering) are auto-fixed. Output shows remaining violations if any.

- [ ] **Step 5: If violations remain, fix them manually**

Re-run `uv run ruff check .` to see remaining issues. Fix each one manually in the source file. Common patterns:
- `F401` unused import → remove the import line
- `E501` line too long → wrap the line

Re-run `uv run ruff check .` until it exits 0.

- [ ] **Step 6: Run tests to confirm nothing broke**

```bash
uv run pytest
```

Expected: All tests pass.

- [ ] **Step 7: Commit**

```bash
git add -u
git commit -m "style: apply ruff formatting and lint fixes"
```

### Task 3: Create pre-commit configuration

**Files:**
- Create: `.pre-commit-config.yaml`

- [ ] **Step 1: Create `.pre-commit-config.yaml`**

```yaml
repos:
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.11.12
    hooks:
      - id: ruff
        args: [--fix]
      - id: ruff-format
```

`ruff` hook runs lint with auto-fix. `ruff-format` hook enforces formatting. Both run on every commit.

- [ ] **Step 2: Install the pre-commit hooks into the local git repo**

```bash
uv run pre-commit install
```

Expected:
```
pre-commit installed at .git/hooks/pre-commit
```

- [ ] **Step 3: Run hooks against all files to verify they pass cleanly**

```bash
uv run pre-commit run --all-files
```

Expected: Both `ruff` and `ruff-format` show `Passed`.

- [ ] **Step 4: Commit**

```bash
git add .pre-commit-config.yaml
git commit -m "chore: add pre-commit hooks for ruff lint and format"
```

### Task 4: Create GitHub Actions workflow

**Files:**
- Create: `.github/workflows/pr-checks.yml`

- [ ] **Step 1: Create the `.github/workflows/` directory and workflow file**

```yaml
name: PR Checks

on:
  pull_request:
    branches: [master]
  push:
    branches: [master]

jobs:
  checks:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.11"

      - name: Install uv
        uses: astral-sh/setup-uv@v5

      - name: Install dependencies
        run: uv sync --group dev

      - name: Lint with ruff
        run: uv run ruff check .

      - name: Check formatting with ruff
        run: uv run ruff format --check .

      - name: Run tests
        run: uv run pytest
```

- [ ] **Step 2: Commit**

```bash
git add .github/workflows/pr-checks.yml
git commit -m "ci: add GitHub Actions workflow for PR checks"
```

- [ ] **Step 3: Push the branch and open a PR to verify the workflow triggers**

```bash
git push origin feature/pipeline-hardening
```

Then open a PR on GitHub targeting `master`. Confirm the `PR Checks` workflow appears and all steps go green.
